WHAT DOES THIS PROJECT DO?
---------------------------
This is a real-time patient monitoring system that uses AI to detect health 
problems automatically. It watches patient vital signs (like heart rate, oxygen 
levels, temperature) and alerts doctors when something is wrong.


HOW IT WORKS (SIMPLE):
----------------------

1. PATIENT SIMULATOR
   - Generates fake patient data (like a real patient's heart monitor)
   - Creates realistic ECG (heart) signals
   - Sends data every 2 seconds

2. AI BRAIN (Machine Learning)
   - Uses a smart AI model (RandomForest) trained on real ECG data
   - Analyzes the patient's heart signals
   - Decides: Is the patient NORMAL or CRITICAL?
   - Accuracy: 98.4% (very accurate!)

3. EMAIL ALERTS (When Critical)
   - Automatically sends email to doctor
   - Professional HTML formatted email
   - Includes all patient vital signs
   - Instant notification within seconds

4. BACKEND SERVER
   - Receives data from the simulator
   - Stores patient information in a database
   - Provides data to the dashboard through an API

5. DASHBOARD (Website)
   - Beautiful dark-themed website
   - Shows real-time charts of patient vitals
   - Displays alerts when patients are in danger
   - Doctors can see all patients at once
   - Login system for security

TECHNICAL DETAILS (FOR DEVELOPERS):
------------------------------------

BACKEND:
- Language: Python
- Framework: Flask
- Database: SQLite
- API: REST API with JSON

MACHINE LEARNING:
- Algorithm: RandomForest Classifier
- Features: ECG signal statistics (mean, std, min, max, median)
- Training Data: 200 samples (147 normal, 53 abnormal)
- Accuracy: 98.4%

EMAIL NOTIFICATIONS:
- Service: Gmail SMTP
- Format: Professional HTML emails
- Trigger: Automatic on critical patient detection
- Content: Patient ID, status, all vitals, timestamp

FRONTEND:
- Framework: React 18 + TypeScript
- Build Tool: Vite
- Charts: Recharts library
- Icons: Lucide React
- Styling: Custom CSS (2500+ lines)

SIMULATOR:
- Generates realistic patient data
- ECG-based feature extraction
- Sends data via HTTP POST
- Updates every 2 seconds


HOW TO RUN THE PROJECT:
------------------------

EASY WAY:
1. Double-click "start.bat"
2. Wait for all services to start
3. Browser opens automatically at http://localhost:5173

MANUAL WAY:
1. Open 3 terminals

   Terminal 1 (Backend):
   cd cloud_server
   python app.py

   Terminal 2 (Simulator):
   python ecg_simulator.py

   Terminal 3 (Frontend):
   cd frontend-react
   npm run dev

2. Open browser: http://localhost:5173


WHAT HAPPENS WHEN YOU RUN IT:
------------------------------

1. Backend starts on port 5000
2. Simulator starts generating patient data
3. Email service initializes (ready to send alerts)
4. Frontend starts on port 5173
5. You see the home page
6. Click "Get Started" or "Sign In"

REAL-WORLD USE CASE:
---------------------

Imagine a hospital with 100 patients. Instead of nurses checking each patient 
every hour, this system:

1. Monitors ALL patients continuously (24/7)
2. Automatically detects problems using AI
3. Sends instant email alerts to doctors when someone is in danger
4. Shows all patient data on one screen
5. Saves lives by catching problems early

Example: A patient's heart rate suddenly jumps to 165 bpm and oxygen drops to 
88%. The AI detects this is critical. Within 2 seconds:
- Alert appears on dashboard
- Email sent to doctor's phone/computer
- Doctor can respond immediately

This is especially useful for:
- ICU (Intensive Care Unit)
- Post-surgery monitoring
- Elderly care facilities
- Remote patient monitoring
- Emergency rooms
- Home healthcare
